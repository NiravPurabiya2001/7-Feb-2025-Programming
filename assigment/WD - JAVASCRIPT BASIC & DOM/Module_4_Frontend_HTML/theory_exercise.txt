Module 4 - Frontend HTML

THEORY EXERCISE

Q1. Define HTML. What is the purpose of HTML in web development?
Answer: HTML stands for HyperText Markup Language. It is the standard language used to create and design webpages. The purpose of HTML is to provide the basic structure or "skeleton" of a webpage, defining where headings, paragraphs, images, and links should go.

Q2. Explain the basic structure of an HTML document. Identify the mandatory tags and their purposes.
Answer: The basic structure of an HTML document includes:
- <!DOCTYPE html>: Tells the browser this is an HTML5 document.
- <html>: The root element that wraps all the content on the page.
- <head>: Contains meta-information about the document like its title and links to CSS files. This content isn't visible on the page.
- <title>: Sets the title of the page shown in the browser tab.
- <body>: Contains all the visible content of the webpage, like text, images, and buttons.

Q3. What is the difference between block-level elements and inline elements in HTML? Provide examples of each.
Answer: 
- Block-level elements always start on a new line and take up the full width available (stretching from left to right). Examples: <div>, <h1> to <h6>, <p>.
- Inline elements do not start on a new line and only take up as much width as necessary for their content. Examples: <span>, <a>, <img>.

Q4. Discuss the role of semantic HTML. Why is it important for accessibility and SEO? Provide examples of semantic elements.
Answer: Semantic HTML uses tags that clearly describe their meaning to both the browser and the developer (e.g., <header>, <footer>, <article> instead of just <div>). It is important for accessibility because screen readers rely on these tags to help visually impaired users navigate the page. It's important for SEO because search engines use them to better understand the content of the page.

Q5. What are HTML forms used for? Describe the purpose of the input, textarea, select, and button elements.
Answer: HTML forms are used to collect data from users, like a login or contact form.
- <input>: A single-line field where users can type data (like text, email, or password).
- <textarea>: A multi-line field where users can type longer text (like a message or comment).
- <select>: Creates a dropdown list from which users can select an option.
- <button>: A clickable element used to submit the form or trigger an action.

Q6. Explain the difference between the GET and POST methods in form submission. When should each be used?
Answer: 
- GET appends the form data to the URL, making it visible. It should be used for simple, non-sensitive requests like search queries.
- POST sends the data securely behind the scenes without showing it in the URL. It should be used when sending sensitive information, like passwords or large amounts of data.

Q7. What is the purpose of the label element in a form, and how does it improve accessibility?
Answer: The <label> element defines a label for form fields. When a user clicks on the text inside the <label>, it automatically focuses or activates the connected input field. This greatly improves accessibility because screen readers will read out the label when the user is on the input field, telling them exactly what to type.

Q8. Explain the structure of an HTML table and the purpose of each of the following elements: <table>, <tr>, <th>, <td>, and <thead>.
Answer: 
- <table>: The container for the entire table.
- <thead>: Groups the header content in the table.
- <tr> (Table Row): Defines a single row in the table.
- <th> (Table Header): Defines a header cell, usually bold and centered.
- <td> (Table Data): Defines a standard cell containing data.

Q9. What is the difference between colspan and rowspan in tables? Provide examples.
Answer: 
- colspan makes a cell stretch horizontally across multiple columns. Example: <td colspan="2"> spans 2 columns.
- rowspan makes a cell stretch vertically across multiple rows. Example: <td rowspan="3"> spans 3 rows.

Q10. Why should tables be used sparingly for layout purposes? What is a better alternative?
Answer: Tables were originally meant only for displaying tabular data, not for designing the layout of a webpage. Using tables for layout makes the HTML code messy, hard to maintain, and bad for screen readers (poor accessibility). A better alternative is using CSS features like Flexbox or CSS Grid to create responsive layouts.
