Module 2 - Lab Exercise

Task: Explain phases of SDLC life cycle.

The Software Development Life Cycle (SDLC) consists of 6 main phases:

1. Requirement Analysis:
In this first phase, the team meets with clients and stakeholders to understand what exactly needs to be built. They gather all the requirements, define the purpose of the software, and figure out who the end-users will be.

2. Planning & Design:
Once the requirements are clear, the system architects and designers plan how to build the software. They choose the programming languages, databases, and create the overall system architecture, wireframes, and UI designs.

3. Implementation (Coding):
This is where the actual programming happens. Developers take the design documents and start writing code to build the application. The work is usually divided into smaller modules or units.

4. Testing:
After the code is written, it is tested to find any bugs or defects. Quality Assurance (QA) testers run various tests to ensure the software works exactly as expected and meets all the original requirements.

5. Deployment:
Once the software is thoroughly tested and approved, it is deployed to the production environment, meaning it is made available to the actual end-users to use.

6. Maintenance:
The software cycle doesn't end after deployment. In the maintenance phase, developers fix any new bugs discovered by users, make performance improvements, and add new features as needed over time.
