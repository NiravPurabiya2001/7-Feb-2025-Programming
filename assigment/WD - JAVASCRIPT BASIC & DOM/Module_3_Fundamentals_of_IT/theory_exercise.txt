Module 3 - Fundamentals of IT

THEORY EXERCISE

Q1. Explain in your own words what a program is and how it functions.
Answer: A program is a set of step-by-step instructions that tells a computer what to do. It functions by taking in some input, processing that input using logic written by a programmer, and producing an output or result.

Q2. What are the key steps involved in the programming process?
Answer: The key steps are:
1. Understanding the problem.
2. Designing a solution (using algorithms or flowcharts).
3. Writing the code (programming).
4. Testing the code for bugs.
5. Debugging and fixing the code.
6. Maintaining and updating the program.

Q3. What are the main differences between high-level and low-level programming languages?
Answer: High-level languages (like Python or JavaScript) are closer to human languages, making them easier to read, write, and understand. Low-level languages (like Assembly or Machine Code) are closer to computer hardware; they are harder for humans to read but run much faster because the computer understands them natively.

Q4. Describe the roles of the client and server in web communication.
Answer: The client (usually a web browser) is the one asking for information. The server is a powerful computer that stores the information and sends it back to the client.

Q5. Explain the function of the TCP/IP model and its layers.
Answer: The TCP/IP model is the set of rules that allows computers to connect and communicate over the internet. Its main layers are:
1. Network Access (handles physical connection)
2. Internet (routes data to the correct destination via IP)
3. Transport (ensures data arrives intact via TCP)
4. Application (provides network services to apps, like HTTP or FTP).

Q6. Explain Client Server Communication
Answer: In client-server communication, a client sends a request over the internet to a server. The server processes this request, gathers the required data, and sends a response back to the client.

Q7. How does broadband differ from fiber-optic internet?
Answer: Broadband typically uses copper wires or cable TV lines to transmit data, which can be affected by distance and heavy usage. Fiber-optic internet uses thin strands of glass to transmit data as pulses of light, making it significantly faster and more reliable than traditional broadband.

Q8. What are the differences between HTTP and HTTPS protocols?
Answer: HTTP sends data in plain text, meaning anyone intercepting it can read it. HTTPS adds an encryption layer (using SSL/TLS), so the data is scrambled and secure.

Q9. What is the role of encryption in securing an application?
Answer: Encryption transforms readable data into an unreadable format. This ensures that even if hackers steal the data, they cannot understand or use it without the secret decryption key.

Q10. What is the difference between system software and application software?
Answer: System software (like Windows or macOS) manages the computer's hardware and provides a platform for other software to run. Application software (like Microsoft Word or a web browser) is designed to help the user perform specific tasks.

Q11. What is the significance of modularity in software architecture?
Answer: Modularity means breaking a large software system into smaller, independent parts (modules). It makes the software easier to understand, develop, test, and maintain because developers can work on individual modules without breaking the whole system.

Q12. Why are layers important in software architecture?
Answer: Layers separate different concerns in an application (e.g., presentation, business logic, data access). This makes the code organized, easier to scale, and allows developers to change one layer without affecting the others.

Q13. Explain the importance of a development environment in software production.
Answer: A development environment gives programmers a safe space to write and test code without affecting the live (production) software that actual users are using. It prevents bugs from crashing the live app.

Q14. What is the difference between source code and machine code?
Answer: Source code is written by programmers in human-readable languages like Java or Python. Machine code consists of 1s and 0s that the computer's processor actually understands and executes.

Q15. Why is version control important in software development?
Answer: Version control (like Git) tracks every change made to the code. It allows developers to revert to older versions if a mistake is made, and it makes teamwork possible by merging code from multiple developers smoothly.

Q16. What are the benefits of using GitHub for students?
Answer: It provides a platform to host personal projects, build a portfolio, collaborate on group assignments, and learn industry-standard version control. They also get free developer tools via the GitHub Student Developer Pack.

Q17. What are the differences between open-source and proprietary software?
Answer: Open-source software (like Linux) has its source code publicly available for anyone to inspect, modify, and distribute, usually for free. Proprietary software (like Microsoft Office) is owned by a company, the source code is hidden, and you usually have to pay to use it.

Q18. How does GIT improve collaboration in a software development team?
Answer: Git allows multiple developers to work on the exact same project simultaneously without overwriting each other's work. They can create branches to build features independently and then merge them into the main project.

Q19. What is the role of application software in businesses?
Answer: It helps businesses increase productivity, manage operations, track finances, and communicate effectively (e.g., accounting software, CRM systems, email clients).

Q20. What are the main stages of the software development process?
Answer: Requirement analysis, Planning/Design, Implementation (Coding), Testing, Deployment, and Maintenance.

Q21. Why is the requirement analysis phase critical in software development?
Answer: Because if you don't fully understand what the client wants at the very beginning, you will end up building the wrong software, wasting time and money.

Q22. What is the role of software analysis in the development process?
Answer: It breaks down the requirements into actionable technical tasks and helps developers understand how the system should function logically.

Q23. What are the key elements of system design?
Answer: Architecture design, database design, user interface (UI) design, and defining the system interfaces and data structures.

Q24. Why is software testing important?
Answer: It ensures the software works correctly, is secure, and provides a good user experience by finding and fixing bugs before the software reaches the customer.

Q25. What types of software maintenance are there?
Answer: Corrective (fixing bugs), Adaptive (updating for new environments/OS), Perfective (adding new features), and Preventive (improving code structure to prevent future issues).

Q26. What are the key differences between web and desktop applications?
Answer: Web applications run in a browser and require an internet connection, but don't need installation. Desktop applications are installed directly on a computer and can often run offline.

Q27. What are the advantages of using web applications over desktop applications?
Answer: They are accessible from any device with a browser, require no installation, and update automatically for all users at once.

Q28. What role does UI/UX design play in application development?
Answer: UX (User Experience) ensures the app is easy and intuitive to use, while UI (User Interface) ensures it is visually appealing. Good UI/UX keeps users engaged and happy.

Q29. What are the differences between native and hybrid mobile apps?
Answer: Native apps are built specifically for one platform (e.g., Swift for iOS) and offer the best performance. Hybrid apps are built once using web technologies (HTML, CSS, JS) and run on multiple platforms, but may not be as fast.

Q30. What is the significance of DFDs in system analysis?
Answer: Data Flow Diagrams (DFDs) visually map out how information moves through a system, helping developers see where data comes from, how it's processed, and where it goes.

Q31. What are the pros and cons of desktop applications compared to web applications?
Answer: Pros: They can run offline, often perform better, and have full access to computer hardware. Cons: They require manual installation and updates, and they usually only work on specific operating systems.

Q32. How do flowcharts help in programming and system design?
Answer: Flowcharts visually represent the logic of a program using shapes and arrows. They make complex logic easier to understand and help programmers plan their code before actually writing it.
